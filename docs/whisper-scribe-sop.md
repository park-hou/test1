# Whisper + Scribe v2 녹취 작업표준

한의학 강의 동영상의 녹취 정확도를 높이되, 두 엔진을 함께 쓰면서 생길 수 있는 운영 혼선을 줄이기 위한 표준 운영 문서다.

## 1. 목적

- Whisper와 Scribe v2를 함께 사용할 때의 기본 운영 원칙을 고정한다.
- 한의학 전문용어, 처방명, 본초명, 혈자리, 숫자 정보의 오류를 줄인다.
- 대량 작업에서도 사람이 봐야 할 구간만 좁혀서 검수 효율을 높인다.
- 결과물을 `raw`와 `clean`으로 분리해 보관한다.

## 2. 이 표준의 기본 판단

두 엔진을 동시에 돌리면 정확도가 오를 수는 있지만, 운영 기준이 없으면 오히려 혼란이 커진다. 따라서 기본 표준은 아래와 같다.

### 기본 표준: single-primary dual-check

1. **Whisper를 1차 기본 엔진으로 사용한다.**
2. **Scribe v2는 2차 검증/보강 엔진으로 사용한다.**
3. 전 구간을 무조건 양쪽으로 다 돌리는 것은 파일럿 또는 고위험 강의에서만 허용한다.
4. 본 운영에서는 **Whisper 전체 수행 -> 플래그 구간만 Scribe 재처리**를 기본값으로 한다.
5. 사람이 두 개의 전체 녹취본을 통으로 비교하지 않는다. **시간 정렬된 충돌 구간만 본다.**

이 방식의 목적은 "둘을 같이 쓴다"가 아니라, **"한 엔진을 기본으로 두고 다른 엔진을 근거 있는 재판정 도구로 쓴다"**는 점에 있다.

## 2A. 혼란 방지 원칙

두 엔진을 함께 쓰는 이유는 정확도 향상이지, 검수자의 판단 부담을 늘리기 위함이 아니다. 따라서 아래 원칙을 운영 중 계속 확인한다.

1. **비교 단위는 전체 문서가 아니라 시간 정렬된 구간이다.**
2. **검수자는 "더 그럴듯한 문장"이 아니라 "사전, 숫자, 강의자료와 더 잘 맞는 근거"를 본다.**
3. **사람이 고르기 어려운 충돌은 억지로 자동 확정하지 않고 보류한다.**
4. **Scribe v2는 기본적으로 Whisper를 뒤엎는 엔진이 아니라, 플래그 구간의 재판정 근거다.**

## 2B. dual-engine 유지/중단 기준

파일럿 5~10개를 돌린 뒤 아래 기준으로 계속 사용할지 판단한다.

### dual-engine 유지

다음 중 2개 이상을 만족하면 유지한다.

- 중요 용어 정확도가 Whisper 단독보다 의미 있게 개선됨
- 숫자/혈자리/처방명 오류 탐지율이 높아짐
- 사람이 최종 확정하기 쉬운 충돌 형태가 반복적으로 관찰됨
- 검수 시간 증가보다 정확도 이득이 더 큼

### dual-engine 축소

다음 중 하나라도 크면 축소한다.

- Scribe 재처리 구간의 상당수가 Whisper와 실질적으로 동일함
- 검수자가 두 결과 중 어느 쪽이 맞는지 판단할 근거를 자주 찾지 못함
- 용어 정확도 개선은 미미한데 충돌 구간만 크게 늘어남

이 경우 운영 모드를 `Whisper 전체 + 고위험 구간만 Scribe`로 더 보수적으로 조정한다.

### dual-engine 중단

다음 조건이면 일단 중단하고 Whisper 단독 + 사전 강화 체계로 되돌린다.

- 중요 용어 정확도 개선이 거의 없고
- 검수 시간과 혼선은 확실히 증가하며
- Scribe 결과가 강의자료/사전보다 우수한 근거를 안정적으로 보여주지 못함

중단은 실패가 아니라, 파일럿의 목적에 따라 더 단순한 운영 모델을 채택하는 결정이다.

## 3. 적용 범위

- 한의학 이론 강의
- 본초학, 방제학, 침구학, 진단학, 생리/병리 강의
- 질의응답이 포함된 세미나형 강의
- 강의 자료(PPT/PDF)가 있거나 없는 동영상 모두

## 5. 운영 모드

### A. 파일럿 모드

다음 상황에서 사용한다.

- 처음 5~10개 강의로 성능을 비교할 때
- 용어사전 초안을 검증할 때
- 강사별 음성 특성이 크게 다를 때

처리 방식:

- Whisper 전체 수행
- Scribe v2 전체 수행
- 구간별 정렬 후 차이 분석
- CER, 용어 정확도, 숫자 정확도 기준으로 정책 확정

### B. 본 운영 모드

기본 권장 모드다.

처리 방식:

- Whisper 전체 수행
- 자동 플래그 생성
- 플래그 구간만 Scribe v2 재처리
- 병합 규칙에 따라 확정
- 충돌이 남는 구간만 사람 검수

### C. 고위험 공개본 모드

다음 경우에만 사용한다.

- 외부 공개용 자막
- 교재/출판 전환용 원고
- 법적 또는 공식 기록 보관 목적

처리 방식:

- Whisper 전체 수행
- Scribe v2 전체 수행
- 전 구간 비교 병합
- 중요 용어 구간 전수 검수

## 7. 작업 전 준비

강의 1개를 시작하기 전에 다음을 확보한다.

- 원본 동영상
- 강의명
- 강사명
- 과목명
- 회차 정보
- 강의 자료(PPT, PDF, 교안) 유무
- 예상 핵심 용어 목록

가능하면 강의 자료에서 텍스트를 먼저 뽑아 용어 후보를 만든다.

## 8. 파일명 규칙

`lecture_id`는 아래 형식으로 고정한다.

`coursecode_instructor_YYYYMMDD_sessionNN`

예시:

`fangji_kim_20260424_session03`

같은 강의에 대한 모든 산출물은 같은 `lecture_id`를 사용한다.

## 9. 용어사전 버전 고정 규칙

혼란을 막기 위해 사전은 배치 단위로 고정한다.

- 배치를 시작할 때 사전 버전을 확정한다. 예: `glossary v0.3`
- 동일 배치 안에서는 사전을 중간에 바꾸지 않는다.
- 중간 수정이 불가피하면 다음처럼 남긴다.
  - 변경일
  - 변경자
  - 추가/수정 이유
  - 어떤 강의부터 적용하는지

## 10. 전처리 표준

### 10.1 오디오 추출

- 포맷: `wav`
- 샘플레이트: `16kHz`
- 채널: `mono`

### 10.2 분할

- 기준 길이: 5~15분
- 구간 간 overlap: 1~2초
- 무음 기준으로 자르되, 문장 중간 절단은 최소화

### 10.3 전처리 체크

아래 항목은 처리 전 확인한다.

- 잡음이 과도한지
- 음량이 너무 작은지
- 좌우 채널 편차가 큰지
- 질문자 음성이 지나치게 작은지
- 강의 중 음악, 박수, 영상 효과음이 많은지

## 11. 1차 녹취: Whisper

Whisper는 기본 엔진이다.

권장 원칙:

- 언어는 한국어로 고정
- word timestamp 사용
- 무음 구간 hallucination 억제 옵션 사용
- 구간별 confidence 또는 유사 지표를 보관

Whisper 결과는 바로 최종본으로 확정하지 않고, 아래 플래그 규칙에 따라 재판정 후보를 뽑는다.

## 12. 자동 플래그 규칙

다음 구간은 자동으로 `review_candidate` 또는 `scribe_rerun`으로 보낸다.

### 12.1 용어 관련

- 공통/과목/강의 사전의 중요 용어가 누락됨
- 발화상 중요 용어가 나와야 하는데 비슷한 일반어로 바뀜
- 사전에 없는 신규 전문용어가 반복 출현함

### 12.2 숫자 관련

- 용량, 횟수, 날짜, 페이지, 장 번호, 혈자리 코드가 포함됨
- 숫자와 단위가 붙는 구간
- 아라비아 숫자와 한글 숫자 혼용이 심한 구간

### 12.3 품질 관련

- confidence가 내부 기준 이하
- 반복문장이나 비정상적 중복이 나타남
- 무음 또는 매우 긴 정적 구간인데 텍스트가 길게 생성됨
- 문장부호와 띄어쓰기가 지나치게 붕괴됨

### 12.4 강의 구조 관련

- 질의응답 구간
- 겹쳐 말하기 구간
- 마이크에서 멀어진 발화
- 판서/슬라이드 읽기 구간

## 13. 2차 재판정: Scribe v2

Scribe v2는 **검증/보강 엔진**으로 사용한다.

기본 규칙:

- 본 운영에서는 플래그 구간만 재처리한다.
- 파일럿 또는 고위험 공개본에서는 전체 구간을 재처리할 수 있다.
- keyterm prompting에는 **해당 배치의 고정된 용어사전**만 사용한다.

### Scribe v2에 넣을 keyterm 기준

다음만 우선 넣는다.

- 처방명
- 본초명
- 혈자리
- 경락명
- 고전명
- 과목 핵심 용어
- 해당 강의 PPT/PDF에서 추출된 주요 용어

무작정 많은 단어를 넣기보다, **정말 중요한 단어를 우선순위별로 넣는다.**

## 14. 병합 원칙

두 결과를 문서 전체 단위로 섞지 말고, **시간 정렬된 구간 단위로만** 결정한다.

### 14.1 자동 승인

다음 조건이면 자동 승인 가능하다.

- 두 엔진 결과가 실질적으로 동일함
- 핵심 용어 차이가 없음
- 숫자 차이가 없음
- 반복/무음 hallucination 징후가 없음

### 14.2 사전 우선 승인

다음 조건이면 사전을 따르는 쪽을 우선 채택한다.

- 한 엔진만 사전의 canonical term과 정확히 일치함
- 다른 엔진은 유사 발음의 일반어 또는 띄어쓰기 깨짐 형태임

예:

- Whisper: `육미지황당`
- Scribe: `육미지황탕`
- 사전 canonical: `육미지황탕`

이 경우 Scribe 결과를 우선 채택한다.

### 14.3 무조건 사람 검수

다음은 자동 승인 금지다.

- 숫자 불일치
- 용량 불일치
- 횟수 불일치
- 혈자리 코드 불일치
- 두 엔진이 서로 다른 처방명을 제시함
- 강의 자료 용어와 둘 다 다름

## 15. 사람 검수 우선순위

### 우선순위 A

- 처방명
- 본초명
- 혈자리
- 경락명
- 숫자와 단위
- 고전 인용
- 두 엔진이 크게 다른 구간

### 우선순위 B

- 긴 문장인데 문장 경계가 무너진 구간
- 영어/약어 혼용 구간
- 학생 질문 구간
- 발화 속도가 매우 빠른 구간

## 16. raw와 clean의 분리

최종 결과는 반드시 두 가지로 남긴다.

### raw

- 실제 발화를 최대한 보존
- 추임새, 반복, 말버릇을 유지할 수 있음
- 연구/보관용 기준본

### clean

- 읽기 편하게 정리한 버전
- 문장부호와 띄어쓰기 정리
- 불필요한 군더더기 제거 가능
- 단, 의미 수정 금지

clean을 만들 때도 원문 의미를 바꾸는 재서술은 금지한다.

## 17. 금지 사항

다음은 혼란을 크게 만들기 때문에 금지한다.

1. 두 엔진 전체 결과를 사람이 통으로 눈대중 비교하는 방식
2. 사전 버전을 배치 도중 몰래 변경하는 것
3. 숫자/용량/혈자리 차이를 자동 승인하는 것
4. LLM이 두 녹취본을 보고 마음대로 문장을 새로 쓰게 하는 것
5. raw와 clean을 한 파일에서 섞어 관리하는 것

## 18. 파일럿 평가 기준

처음 5~10개 강의에서는 아래를 기록한다.

- CER
- 중요 용어 정확도
- 숫자 정확도
- 자동 승인 비율
- 사람 검수 소요 시간
- 강사별 오류 패턴

추천 내부 목표:

- 중요 용어 정확도: 매우 높게 유지
- 숫자 정확도: 사람이 확인한 최종본 기준으로 100% 확정
- 자동 승인 비율: 무리하게 높이지 말고 보수적으로 시작

## 20. 운영 권장안

현재 단계에서의 권장안은 다음과 같다.

1. **파일럿 10개는 Whisper 전체 + Scribe 전체**
2. 파일럿 결과로 플래그 기준을 조정
3. 본 운영은 **Whisper 전체 + 플래그 구간만 Scribe**
4. 중요 강의나 공개본만 전 구간 dual-pass

이렇게 해야 "둘을 함께 써서 이득은 취하되, 둘 때문에 작업 흐름이 무너지는 문제"를 줄일 수 있다.
